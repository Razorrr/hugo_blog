WebSocket

Websocket是为了解决client和service之间实现双向长连接通信而设计的。主要应用于监控、聊天。websocket与http都是基于TCP协议，但与http的区别在于，http是一种单向无状态协议，而websocket是一种双向有状态协议。

RESTful

REST – REpresentational State Transfer,全称是 Resource Representational State Transfer, 直接翻译：资源在网络中以某种表现形式进行状态转移。一句话总结:URL定位资源，用HTTP动词（method:GET,POST,DELETE,DETC）描述操作。RESTful API可以通过一套统一的接口为Client提供服务。

RESTful API原则

  • url简洁
  • 只带名词不带动词
  • 保证GET方法安全，使资源不被污染

MVC

主要是为了实现前后端逻辑分离。 * M(Model):后台抽象逻辑 * V(View): 状态等综合形成的上下文 * C(Control):用户端信息
