Terminal代理设置导致pip和git无法正常使用

2017-11-16

问题描述

今天用pip安装东西的时候忽然频繁出现连不上的问题，百思不得其解，尝试了用各种代理各种恢复未果。然后用git clone出现如下错误：

Cloning into 'spider'...
fatal: unable to access 'https://github.com/gsh199449/spider.git/': Failed to connect to 127.0.0.1 port 44131: Connection refused

才怀疑到环境上。

解决办法

先在终端中查看当前代理状况

env | grep -i proxy

然后发现是这样的：

HTTP_PROXY=http://127.0.0.1:44131/
http_proxy=http://127.0.0.1:44131/
no_proxy=localhost,127.0.0.0/8,::1
NO_PROXY=localhost,127.0.0.0/8,::1
HTTPS_PROXY=http://127.0.0.1:44131/

故可以确定是代理设置的问题，所以将其恢复为无代理状态。

unset http_proxy

或者

http_proxy=""

至此，问题解决。
